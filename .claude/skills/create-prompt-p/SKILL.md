# create-prompt-p

niji journey用プロンプトを生成するスキル（パーソナライズ版）。

## トリガー
- ユーザーが `/create-prompt-p` を実行したとき
- ユーザーが「パーソナライズ版で」「personalize」等を指定したとき

## 説明
入力情報からniji journey v7向けの画像生成プロンプトを生成する。
パーソナライズ版は niji journey の「パーソナライズ」機能を前提とした画風で生成する。

非パーソナライズ版（`/create-prompt`）との違い：
- 顔周辺のみ高密度、他は極端に情報密度を落とす
- 線がよりラフで途切れが多い
- 水彩/インク的な「紙に染み込む」質感
- 紙の地（白）が大きく残ることを許容する
- ちびキャラ〜極端な低頭身もOK

## 入力
- スキル実行時の引数テキスト
- または既存の履歴ファイルパス（再開時）

## 実行手順

### 1. 入力の読み取り
- 引数テキストを読む
- 既存履歴ファイルが渡された場合は再開モード

### 1.5. キャラクター設定の参照
- 入力テキストに含まれる名前が `characters/` 配下のMDファイル名と一致するか確認する
  - 例: 「しずくaiが団子を食べている」→ `characters/しずくai.md` を探す
- ファイルが見つかった場合、そのキャラクターの外見設定を入力に統合する
  - キャラクター設定の外見（髪色・目色・衣装等）をプロンプトの素材情報として使用
  - 入力で明示的に上書きされた項目はそちらを優先する
  - キャラクター設定に含まれない情報（構図・背景・雰囲気等）は入力テキストに従う
- ファイルが見つからない場合はスキップ（キャラクター未登録として通常フローへ）
- `_template.md` は除外

### 2. ドメインカプセルの読み込み
以下の順序でカプセルを読む：

**Step 1: 共通カプセル**
`.claude/skills/create-prompt/domains/` 配下の全 `.md` ファイル
（`_template.md`、`personalize/` ディレクトリは除外）

**Step 2: パーソナライズ差分カプセル**
`.claude/skills/create-prompt/domains/personalize/` 配下の全 `.md` ファイル

**Step 3: 上書き適用**
差分カプセルの `overrides` に記載されたカプセルのルールは、
差分カプセルのルールで上書きする。
上書き対象でないルールはそのまま共通カプセルに従う。

#### カプセル構成
| レイヤー | カプセル | 内容 |
|---------|---------|------|
| 共通 | output-rules.md [10] | 合成ルール・概念語・禁止ブロック |
| 共通 | face-expression.md [9] | 顔・表情・目元構造制御 |
| **差分** | **style-override.md [9]** | **画風・線質を水彩/インク寄りに上書き** |
| **差分** | **coloring-override.md [9]** | **白背景→紙の地許容、背景を色面のみに上書き** |
| **差分** | **form-override.md [9]** | **低頭身・ちびキャラ許容に上書き** |
| 共通 | art-style.md [8] | 画風基礎（差分で部分上書き） |
| 共通 | character-form.md [8] | 造形基礎（差分で部分上書き） |
| 共通 | coloring.md [7] | 色彩基礎（差分で部分上書き） |
| 共通 | painterly-texture.md [7] | 質感基礎（差分で部分上書き） |
| 共通 | composition.md [5] | 構図基礎（差分で部分上書き） |
| 共通 | character.md [4] | キャラクター外観基礎 |

### 3. プロンプト合成

#### 優先順（パーソナライズ版固定）
```
0  キャンバス比率
1  主役物理定義
2  焦点
3  髪
4  目
5  肌
6  衣装
7  構図・カメラ
7.5 表情
8  光
9  色面構造
10 影色
11 背景
12 密度差
13 線の役割
14 禁止
```

#### テイスト固定ルール（最重要）
以下を常に成立させる：
- 顔周辺のみ高密度で明確に描写
- それ以外（髪・衣装・背景）は情報密度を落とす
- 線は補助、色面が主役
- 線は途切れや省略を許容し、説明しすぎない
- 塗りは均一にせず、ムラ・にじみ・濃淡差を残す
- 背景は物体として描かず、色面の重なりとして扱う

#### 合成ルール
- output-rules.mdの優先順位（画風成立 > 主役視認性 > 配色成立 > 構造維持 > 情報再現）を常に守る
- 差分カプセルが共通カプセルと矛盾する場合、差分カプセルを優先
- 入力情報はそのまま再現せず再構成する
- 主役を阻害する情報は削減する（削減優先順：装飾→質感→背景→色数→細部）
- 曖昧な雰囲気語・ジャンルラベルは使わず、視覚的に解釈可能な要素へ分解する

### 4〜7. 履歴ファイル・出力・継続対話・再開
`/create-prompt` と同じ仕組みに準拠する。
（→ `.claude/skills/create-prompt/SKILL.md` のステップ4〜7を参照）

履歴ファイル名: `prompts/history/YYYY-MM-DD_HHmm_{title}.md`
（非パーソナライズ版と同じディレクトリに保存。ファイル内で「パーソナライズ版」と明記する）

## 品質チェック（パーソナライズ版）

共通の品質チェック（`/create-prompt` SKILL.mdのA〜D）に加えて、
以下のパーソナライズ固有チェックを行う：

### P. パーソナライズ固有チェック

| # | チェック項目 |
|---|------------|
| P1 | 顔周辺が他の領域より明確に高密度になっているか |
| P2 | 髪・衣装・背景の情報量が顔より落ちているか（ただし形の大枠は読めるか） |
| P3 | 背景が物体として描写されていないか（色面のみか） |
| P4 | 線が荒ぶっていないか（走り書き・スクリブル的な線になっていないか） |
| P5 | 光が顔を分断していないか |
| P6 | 画材名ではなく「見え方」で統一されているか |
| P7 | **形状の判別**: 顔・手・衣装が記号化・省略されすぎていないか（「絵として見れる」状態か） |
| P8 | **密度のグラデーション**: 密度差が急激すぎないか（顔→髪→衣装→背景の順で段階的に落ちているか） |
| P9 | **衣装の識別性**: 衣装の層構造・素材の違いが最低限読み取れるか |
| P10 | **キャラ特徴の維持**: キャラクターの識別要素（ヘッドホン、リボン等）が省略されていないか |

## ガードレール（パーソナライズ版）

共通のガードレール（`/create-prompt` SKILL.md）に加えて：

### 描写品質の維持（最重要）
- パーソナライズ版でも「丁寧に描かれた挿絵」の品質を目指す
- 「ラフスケッチ」「落書き」レベルまで品質を落とさない
- 省略は「描かないことを選んだ」のであって「描けなかった」ように見えない状態を維持
- 色面が流れすぎて何を描いているか分からなくなる状態は修正する

### デフォルメの安全性
- 低頭身・ちびキャラ比率を許容するが、記号的な顔表現は禁止
- 顔の成立条件（目・頬・顎・首の連続面）は常に維持
- 手足が完全に消失・記号化しないよう最低限の形状を維持

### テイストの一貫性
- 画風が水彩/インク寄りから逸脱していないか確認
- 油絵的な厚塗り、デジタルの均一塗りに流れていたら修正
- 逆に荒ぶりすぎ、走り書き的になっていたら制御する
