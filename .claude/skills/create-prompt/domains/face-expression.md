# 顔・表情・目元構造制御

- priority: 9

## ルール

### 顔・表情の基本
- 可愛さや感情を強調しすぎない、中性的で素朴な設計を基本とする
- はっきりしたアイラインや濃い輪郭線は避ける
- 目元は最小限の線、淡い陰影、色のにじみで表現する
- 表情は断定せず、見る側が解釈できる余白を残す
- 顔まわりは、線を強くせずに、明度差と色相差で形を読み取りやすくする
- 目鼻口は線で描き切らず、淡い陰影と色の境界で静かに成立させる

### 目元構造制御

#### 禁止
- 明確なアイライン禁止
- 太い上まつげ線禁止
- 虹彩外周の濃線禁止
- 白目と肌の強い境界禁止
- 記号的まつげ禁止
- 上瞼を単色で縁取らない
- まつげを黒線で描かない

#### 成立条件
- 虹彩は色相差で成立させる
- 上瞼は明度圧縮で示す
- 輪郭線で囲わない
- 誇張なしで表情が読める
- 目は縁取りよりも色相差で成立

### 顔の構築方法
- 顔は色相遷移と明度遷移で構築する
- 線は補助的役割のみ
- 色の境界でエッジを示す

## キーワード
- 顔構造: face defined by hue transitions and soft value shifts, minimal linework around eyes
- 目元: iris constructed by gentle chroma shifts, upper eyelid implied by value compression, no graphic eyeliner
- 表情: understated expression with interpretive space, neutral and unassuming facial design
- 禁止: bold eyeliner, heavy upper lash line, dark iris outline, graphic manga eyes

## 例
- 良い例: "soft light yellow-green eyes constructed by gentle chroma shifts, face defined by hue transitions and soft value shifts, understated expression leaving room for viewer interpretation"
- 悪い例: "big sparkling anime eyes with bold eyeliner"（記号的表現）
