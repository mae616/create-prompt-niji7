# create-prompt-niji7

niji journey v7 向けのイラスト生成プロンプトを、Claude Code のスキルとして構築・管理するプロジェクト。

## 何ができるか

「描きたいもの」を伝えると、niji journey v7 で安定して出力できるプロンプト（英語/日本語）を生成する。

```
/create-prompt 春の公園で団子を食べている猫耳の少女
```

GPT や Gemini のカスタム指示と違い、**ドメインカプセル**（世界観・画風・色彩・顔構造 等）に知識を分離し、矛盾なく合成する仕組みを持つ。

## 思想

### なぜカプセル化するのか

niji journey のプロンプトは、長くなるほど指示同士が干渉する。
GPT のカスタム指示では「全部を1つの指示に詰め込む」ため、画風と色彩の指示が衝突したり、修正が波及したりする。

このプロジェクトでは指示を**ドメインごとに独立したカプセル**（MDファイル）に分離する。

- 1つ変えても他に影響しない
- 優先度（priority）で競合を解決する
- ドメイン単位で追加・削除・差し替えが可能

### カプセルの優先順位

プロンプト生成時は常にこの順序で判断する：

```
画風成立 > 主役の視認性 > 配色成立 > 構造維持 > 情報再現
```

入力情報の忠実な再現よりも、**画面として成立すること**を優先する。

## 2つのモード

| コマンド | 用途 | 画風の方向性 |
|---------|------|------------|
| `/create-prompt` | 非パーソナライズ版 | 水彩×油絵ハイブリッド、色面で空間成立 |
| `/create-prompt-p` | パーソナライズ版 | 水彩/インク寄り、紙の地が残る、ラフスケッチ感 |

パーソナライズ版は niji journey の「パーソナライズ」機能と組み合わせて使う想定。
共通カプセル 9 種を共有し、差分カプセル 3 種で画風・色彩・造形を上書きする。

## 使い方

### 前提

- [Claude Code](https://claude.ai/claude-code) がインストールされていること
- devcontainer で開くか、ローカルで `claude` コマンドが使えること

### プロンプトを生成する

```bash
# 非パーソナライズ版
/create-prompt 猫耳の少女が窓辺で伸びをしている

# パーソナライズ版
/create-prompt-p 猫耳の少女が窓辺で伸びをしている
```

### 修正・研ぎ澄まし

生成後、そのまま会話を続ける。

```
もっと暖色寄りにして
背景の情報量を減らして
表情をもう少し曖昧に
```

修正のたびに v2, v3... と履歴ファイルに追記される。

### 再開

前回の履歴ファイルを渡せば、途中から続けられる。

```
/create-prompt prompts/history/2026-03-24_1430_cat-ear-girl-stretching.md
```

### 完成

```
完成
```

でステータスが「完成」に変わり、そのプロンプトの作業が終わる。

## ディレクトリ構造

```
.
├── .claude/skills/
│   ├── create-prompt/          # 非パーソナライズ版
│   │   ├── SKILL.md            # スキル本体
│   │   └── domains/            # ドメインカプセル（共通 9 種）
│   │       └── personalize/    # パーソナライズ差分（3 種）
│   └── create-prompt-p/        # パーソナライズ版
│       └── SKILL.md            # スキル本体（共通カプセル + 差分を読む）
├── prompts/
│   └── history/                # 生成したプロンプトの全履歴
├── CLAUDE.md                   # プロジェクトルール・ペルソナ
└── .devcontainer/              # devcontainer 設定
```

## ドメインカプセル一覧

### 共通（両モードで使用）

| priority | カプセル | 役割 |
|----------|---------|------|
| 10 | output-rules | 合成の最上位ルール・プロンプト形式・禁止ブロック |
| 9 | face-expression | 顔・表情・目元構造（アイライン禁止、虹彩は色相差で成立） |
| 8 | art-style | 画風・線質・色面構造（線は補助、色面が主役） |
| 8 | character-form | 造形・絵柄ポジション（漫画と絵画の中間） |
| 7 | coloring | 色彩・配色・影色（白背景禁止、影に別色相混合） |
| 7 | painterly-texture | 絵画的質感・画材・技法（水彩×油絵ハイブリッド） |
| 5 | composition | 情報密度・視線誘導・空間設計 |
| 4 | character | キャラクター外観基礎（素材と色相のみ、画風は決めない） |
| 3 | world-setting | 世界観・環境（素材・光・色相傾向） |

### パーソナライズ差分（`/create-prompt-p` でのみ適用）

| カプセル | 上書き対象 | 変更内容 |
|---------|----------|---------|
| style-override | art-style, painterly-texture | 水彩/インク寄り、線のラフさ強化、画材名より「見え方」優先 |
| coloring-override | coloring, composition | 紙の地（白）許容、背景は色面のみ、光は控えめ |
| form-override | character-form | 2〜3 頭身のちびキャラ許容（顔の成立条件は維持） |

## カプセルを編集する

カプセルは `.claude/skills/create-prompt/domains/` にある MD ファイル。
テンプレート（`_template.md`）をコピーして新しいカプセルを作れる。

各カプセルの構造：

```markdown
# {ドメイン名}

## メタデータ
- priority: 5       # 1〜10、高いほど競合時に優先
- conflicts: []      # 競合する可能性のあるカプセル名

## ルール
## キーワード
## 例
```

## 品質チェック

プロンプト生成時、出力前に自動で 18 項目のセルフレビューを通す。

- **カプセル準拠**: 概念語形式、禁止ブロック、画材/技法形式
- **画面成立**: 画風→主役→配色→構造→情報の優先順位
- **禁止事項**: 写実語彙、純黒、ジャンルラベル単体
- **フレーズ構造**: 1 フレーズ 1 意図、順序、衝突

不合格の項目は自動修正してから出力する。
